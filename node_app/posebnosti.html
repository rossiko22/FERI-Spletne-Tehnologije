Posebnosti implementacije - Fitness Buddy

Ta dokument opredeljuje tehnične zahteve, povezane z implementacijo posameznih funkcionalnosti aplikacije Fitness Buddy. Za vsako funkcionalnost so navedene specifične tehnologije, knjižnice ali senzorji, ki so potrebni za njeno delovanje.

1. Beleženje treningov

Tehnične zahteve

  • Lokalna baza podatkov za shranjevanje treningov brez internetne povezave (IndexedDB prek knjižnice Dexie.js)
  • Backend REST API za sinhronizacijo podatkov z oddaljenim strežnikom (Node.js + Express)
  • Relacijska ali dokumentna baza na strežniku (PostgreSQL ali MongoDB)

Senzorji / posebnosti naprave

  • Ni obveznih senzorjev za osnovno delovanje
  • Opcijsko: akcelerometer / žiroskop za samodejno zaznavanje tipa gibanja

2. Sledenje prehrani

Tehnične zahteve

  • Integracija z javno bazo podatkov o živilih (npr. Open Food Facts API) za pridobivanje kalorij in makrohranil
  • Lokalno predpomnjenje pogosto iskanih živil za delovanje brez interneta

Senzorji / posebnosti naprave

  • Opcijsko: kamera za skeniranje črtne kode živila in samodejno iskanje v bazi

3. Postavljanje ciljev

Tehnične zahteve

  • Logika na odjemalcu za izračun napredka in primerjavo s ciljem
  • Shramba ciljev v lokalni bazi (IndexedDB) in sinhronizacija s strežnikom

Senzorji / posebnosti naprave

  • Ni posebnih zahtev glede senzorjev

4. Pametni opomniki

Tehnične zahteve

  • Web Notifications API za prikazovanje obvestil v brskalniku
  • Service Worker za prikaz opomnikov v ozadju, tudi ko aplikacija ni odprta
  • Background Sync API za usklajevanje opomnikov z nastavitvami na strežniku

Senzorji / posebnosti naprave

  • Naprava mora imeti omogočena sistemska obvestila (push notifications)

5. Analiza napredka

Tehnične zahteve

  • Knjižnica za vizualizacijo podatkov na odjemalcu (Chart.js ali D3.js)
  • Agregacija podatkov iz lokalne baze za prikaz grafov brez internetne povezave

Senzorji / posebnosti naprave

  • Ni posebnih zahtev glede senzorjev

6. Sledenje hidraciji

Tehnične zahteve

  • Lokalna shramba dnevnih vrednosti (IndexedDB / localStorage)
  • Integracija z sistemom opomnikov (Service Worker) za periodična opozorila

Senzorji / posebnosti naprave

  • Ni obveznih senzorjev

7. Upravljanje profila

Tehnične zahteve

  • Avtentikacija uporabnika (JWT tokeni ali OAuth 2.0)
  • Varno shranjevanje profilnih podatkov na strežniku
  • Lokalna kopija ključnih nastavitev za offline personalizacijo

Senzorji / posebnosti naprave

  • Ni posebnih zahtev glede senzorjev

8. Offline delovanje (PWA)

Tehnične zahteve

  • Service Worker za predpomnjenje virov aplikacije (HTML, CSS, JS, slike)
  • Web App Manifest za namestitev aplikacije na domači zaslon naprave
  • IndexedDB za lokalno shranjevanje podatkov med offline delom
  • Background Sync API za sinhronizacijo ob vzpostavitvi internetne povezave

Senzorji / posebnosti naprave

  • Ni posebnih senzorjev, vendar mora naprava podpirati Service Worker API (podprt v vseh modernih brskalnikih)

Povzetek tehnološkega sklada

Plast Tehnologija
Frontend / odjemalec HTML, CSS, JavaScript, PWA (Service Worker, Web App Manifest)
Lokalna shramba IndexedDB (Dexie.js), localStorage
Backend Node.js, Express
Baza podatkov PostgreSQL ali MongoDB
Vizualizacija Chart.js ali D3.js
Avtentikacija JWT tokeni
Obvestila Web Notifications API, Background Sync API
Opcijsko (senzorji) Kamera (črtna koda), akcelerometer (zaznavanje gibanja)
